## New feature in detail — SENAITE samples list collapse + Vials/Assigned columns

URL: `http://localhost:3101/#senaite/samples`. Page used to be a flat list mixing parent ARs + secondary ARs (sub-samples) returned 1:1 from SENAITE. Now:

- **Sub-samples filtered server-side** in `/senaite/samples` (Mk1 `main.py:10711`). New `include_sub_samples=False` query param (defaults exclude). Backend over-fetches `limit*2` from SENAITE and trims, so each frontend page hits ~50 parents even when sub-samples cluster in the result.
- **Vials column**: chevron + total vial count (parent + sub-samples) for parents in `lims_samples` that have any sub-samples. Single-vial parents and un-received samples render `—`.
- **Assigned column**: a single full-name role badge (`H HPLC`, `E Endo`, `S Ster`, `X Xtra`, `? Unassigned`) showing the parent AR's own `assignment_role`. Sub-sample roles appear inline in the expanded section, not aggregated on the parent row.
- **Expand-in-row**: chevron click → fetches `/api/sub-samples/{parent}` lazy + cached → renders an inset table beneath the parent showing each sub-sample's ID, vial sequence, role badge, and remarks. Each sub-row is clickable and routes to detail.
- **Sub-sample search hits** (e.g. searching `BW-0010-S03`) bypass the filter and render the sub-sample as a flat row, with its own role badge surfaced via the same `aggregates` endpoint resolving sub-sample IDs against `lims_sub_samples`.
- **PAGE_SIZE bumped 25 → 50** to match the pre-filter visual density.

Backend endpoint added: `POST /api/sub-samples/aggregates` taking `{parent_sample_ids: [...]}`, returning `{aggregates: {sample_id: {vial_count, parent_role}}}`. Auth-gated. 2 tests passing.

## Conventions worth knowing (carried from prior handoff)

- Mk1 backend uses lightweight `_run_migrations()` ALTER statements in `database.py`, NOT Alembic. **As of this session, each statement is wrapped in its own try/except with WARNING-level skip logging.**
- Mk1 ↔ IS API key env var: `ACCU_MK1_API_KEY` (canonical) with fallback to legacy `INTEGRATION_SERVICE_API_KEY`
- All `docker exec accumark_postgres ...` paths need `MSYS_NO_PATHCONV=1` prefix on Git Bash
- Mk1 + IS backends are bind-mounted in dev — code edits live after restart, no rebuild
- **Coabuilder src is NOT bind-mounted** — every code change needs `docker compose up -d --build coabuilder`
- Frontend: Vite HMR auto-picks up changes
- WP: DevKinsta bind-mounted — PHP edits live without restart
