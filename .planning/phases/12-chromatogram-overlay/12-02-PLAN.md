Replace with:
```typescript
const displayChromTraces = useMemo(() => {
  // Start with sample traces, filtered by active analyte if needed
  let sampleTraces: ChromatogramTrace[]
  if (!activeAnalyte || !hasMultipleAnalytes) {
    sampleTraces = chromTraces
  } else {
    const upper = activeAnalyte.toUpperCase()
    const filtered = chromTraces.filter(t => {
      const n = t.name.toUpperCase()
      if (n.includes('BLANK')) return false
      return n.includes(upper)
    })
    sampleTraces = filtered.length > 0
      ? filtered
      : chromTraces.filter(t => !t.name.toUpperCase().includes('BLANK'))
  }

  // Prepend standard reference trace from the active calibration curve (if available)
  // Cast chromatogram_data to Record<string, unknown> because the runtime format
  // may be multi-concentration (keyed by conc level) even though the TS type
  // only declares the old single-trace shape.
  if (selectedCal?.chromatogram_data) {
    const stdTrace = extractStandardTrace(
      selectedCal.chromatogram_data as unknown as Record<string, unknown>,
    )
    if (stdTrace) {
      return [stdTrace, ...sampleTraces]
    }
  }

  return sampleTraces
}, [chromTraces, activeAnalyte, hasMultipleAnalytes, selectedCal])
```

Key design decisions:
- **Standard trace prepended (index 0):** Recharts renders lines in order, so the standard (dashed) renders first and the sample (solid) renders on top. The standard also gets TRACE_COLORS[0] (blue), and sample traces shift to subsequent colors — this is fine because the dashed/opacity styling already distinguishes them.
- **selectedCal dependency:** The memo re-computes when the selected calibration changes (including when switching analyte tabs in blends, since `handleAnalyteChange` updates `selectedCalId` which changes `selectedCal`).
- **Type cast:** The CalibrationCurve TS interface declares `chromatogram_data: { times: number[]; signals: number[] } | null`, but the actual API response may use multi-concentration format (`{"1": {times, signals}, "10": {times, signals}}`). The cast to `Record<string, unknown>` lets `extractStandardTrace` handle both formats correctly.
- **Graceful degradation:** If `chromatogram_data` is null or `extractStandardTrace` returns null, the sample-only traces render as before.
- **No changes to AnalysisResults or ChromatogramChart:** The overlay "just works" because we're passing a styled trace through the existing pipeline.
