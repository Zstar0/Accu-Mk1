<objective>
Parse `_std_` PeakData files from .rslt folders as standard reference injections, separate from sample data.

Purpose: Standard injection files contain same-method reference RTs needed for accurate identity checks on blends. Currently these files are either mixed into sample injections or ignored. This plan separates them and extracts the reference data.

Output: Modified parser that detects `_std_` files, extracts main peak RT per analyte, and returns them in the parse response.
</objective>

  <action>
Modify the parser to detect and separately handle `_std_` PeakData files:

1. **Add a `StandardInjection` dataclass** after the existing `InjectionData` class:
```python
@dataclass
class StandardInjection:
    """Parsed standard injection reference data."""
    analyte_label: str        # e.g. "BPC157", "GHK", "TB17-23"
    main_peak_rt: float       # RT of highest Area% peak
    main_peak_area_pct: float # Area% of the main peak
    source_sample_id: str     # e.g. "P-0111" from "Sample name:" metadata
    filename: str             # original filename for audit
```

2. **Add `standard_injections` field to `HPLCParseResult`**:
```python
standard_injections: list[StandardInjection] = field(default_factory=list)
```

3. **Add a helper function `_is_standard_injection(filename: str) -> bool`** that returns True if `_std_` (case-insensitive) appears in the filename. This distinguishes `PB-0065_Inj_1_std_BPC157_PeakData.csv` from `PB-0065_Inj_1_BPC157_PeakData.csv`.

4. **Add a function `_extract_standard_info(filename: str) -> tuple[str, str]`** that extracts `(analyte_label, "")` from standard filenames. The analyte label is the part between `_std_` and `_PeakData`. Examples:
   - `PB-0065_Inj_1_std_BPC157_PeakData.csv` → analyte_label = "BPC157"
   - `PB-0065_Inj_1_std_GHK_PeakData.csv` → analyte_label = "GHK"
   - `PB-0065_Inj_1_std_TB17-23_PeakData.csv` → analyte_label = "TB17-23"

5. **Add a function `parse_standard_injection(filename: str, content: str) -> StandardInjection`** that:
   - Reuses `parse_peakdata_csv` to parse the peak table (the CSV format is identical)
   - Extracts the main peak RT from the parsed injection (the peak with highest Area%)
   - Extracts `source_sample_id` from the "Sample name:" metadata line in the CSV content (parse the lines after the peak table for a line starting with "Sample name:" — the value is the second CSV field, e.g. "P-0111_Inj_1" → strip to "P-0111" by taking everything before the first "_Inj_")
   - Calls `_extract_standard_info(filename)` for the analyte label
   - Returns a `StandardInjection` instance

6. **Modify `parse_hplc_files`** to:
   - Check each file with `_is_standard_injection(filename)` BEFORE parsing
   - If it's a standard injection: call `parse_standard_injection` and add to a separate `standard_injections` list
   - If it's a regular sample file: process as before (call `parse_peakdata_csv`)
   - Return `HPLCParseResult(injections=injections, errors=errors, standard_injections=standard_injections)`

This ensures standard injection files are NEVER mixed into the regular `injections` list.
