  <name>Task 1: StandardCurveReview component</name>
  <files>src/components/hplc/StandardCurveReview.tsx</files>
  <action>
Create a new component `StandardCurveReview` that displays curve data and allows the user to confirm creation.

Props:
```typescript
interface StandardCurveReviewProps {
  peptideId: number
  samplePrepId: string              // e.g. "P-0136"
  concentrations: number[]          // ug/mL per dilution
  areas: number[]                   // Peak areas per dilution
  rts: number[]                     // Retention times per dilution
  chromatogramData?: { times: number[]; signals: number[] }
  sharepointFolder?: string
  vendor?: string                   // From prep.manufacturer
  notes?: string                    // From prep.standard_notes
  instrument?: string               // From parse result
  onCurveCreated: (curve: CalibrationCurve) => void  // Callback after success
}
```

UI layout (use existing shadcn Card, Badge, Button, Alert components):

1. **Header section:** "Calibration Curve Preview" with a Badge showing n data points
2. **Data table:** Simple table with columns: Concentration (ug/mL) | Peak Area | RT (min). One row per dilution level. Use font-mono for numbers.
3. **Regression stats:** Display computed regression locally for preview:
   - Compute simple linear regression client-side for PREVIEW ONLY (slope, intercept, r-squared). Use the standard formulas: slope = sum((x-mx)(y-my)) / sum((x-mx)^2), etc. This is purely for preview display — the backend recomputes authoritatively.
   - Show: Slope, Intercept, R-squared (4 decimal places), n points
   - If r_squared < 0.99, show an amber warning: "R-squared below 0.99 — review data quality"
   - If fewer than 3 valid points, show error state: "Insufficient data points (need >= 3)"
4. **Provenance info:** Small muted text showing source_sample_id, vendor, instrument
5. **Action button:** "Create Calibration Curve" primary button, disabled when < 3 points or while submitting
   - On click: call `createCalibrationFromStandard(peptideId, { sample_prep_id, concentrations, areas, rts, chromatogram_data, source_sharepoint_folder, vendor, notes, instrument })`
   - Show loading spinner during submission
   - On success: show success alert "Calibration curve created (R-squared: X.XXXX)", call onCurveCreated
   - On error: show error alert with message

Use Zustand selector syntax. No manual useMemo/useCallback (React Compiler handles it). Use existing Tailwind v4 utility classes.

<task type="auto">
  <name>Task 2: Wire standard detection into HPLC flyout</name>
  <files>src/components/hplc/SamplePrepHplcFlyout.tsx</files>
  <action>
Modify SamplePrepHplcFlyout to detect standard preps and show the curve review panel instead of the normal analysis flow.

1. **Import** StandardCurveReview and createCalibrationFromStandard at the top of the file.

2. **Detect standard prep:** The `prep` prop (SamplePrep) already has `is_standard` field. Check `prep.is_standard === true` to branch behavior.

3. **Extract concentration/area data from scan results:**
   When the HPLC scan completes for a standard prep, the flyout already receives parsed injection data (HPLCParseResult with injections). For standards, each injection corresponds to a concentration level.

   After parse results load, map injections to concentration/area/RT triples:
   - The concentration for each injection should come from the prep's `vial_data` JSON (which stores per-dilution target_conc_ug_ml from the wizard). Access via `prep.vial_data` array — each entry has `vial_number` and concentration info.
   - Alternatively, if vial_data concentrations are available, match by injection index to sorted concentrations.
   - For each injection, extract the largest peak area (or the peak matching the expected analyte RT if calibration exists).
   - Collect RT from the same peak.

   Build arrays: `concentrations: number[]`, `areas: number[]`, `rts: number[]`

4. **Extract chromatogram data:** The flyout already loads DAD1A chromatogram files. For standards, pick the first valid chromatogram trace (or the highest concentration injection's trace) to store as `chromatogram_data`.

5. **Render StandardCurveReview:** When `prep.is_standard` and we have concentration/area data:
   - Replace (or show below) the normal analysis results section with `<StandardCurveReview>` passing all extracted data
   - Pass `prep.manufacturer` as vendor, `prep.standard_notes` as notes
   - Pass the HPLC scan match's SharePoint folder as sharepointFolder
   - Pass `prep.senaite_sample_id || prep.sample_id` as samplePrepId

6. **onCurveCreated callback:** After curve creation succeeds, show a success state in the flyout. Optionally invalidate/refresh calibration data if the flyout displays it.

IMPORTANT: The normal (non-standard) flyout flow must be completely unaffected. The standard branch is purely additive — only activates when `prep.is_standard === true`. Keep the existing analysis auto-run, calibration selection, and results display intact for production preps.

Do NOT compute regression on the frontend for the actual curve creation — only for preview display in StandardCurveReview. The backend is the source of truth for regression computation.
  </action>
