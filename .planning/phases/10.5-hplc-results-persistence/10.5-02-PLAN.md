**Sub-change B: DB-first load with dual-effect guard.**

The flyout has TWO separate effects that matter:
1. **Reset effect** (~line 522): `useEffect(() => { if (open) { /* resets all state */ } }, [open, prep.id])` — synchronous, resets state.
2. **SharePoint load effect** (~line 583): `useEffect(() => { if (open) loadPeakData() }, [open, loadPeakData])` — triggers the async SharePoint scan via `loadPeakData` useCallback (~line 543).

These are independent effects. Returning early from one does NOT stop the other. The fix requires guarding `loadPeakData` itself.

**Step B1: Modify the reset effect (~line 522) to add an async DB check.**

After the existing synchronous resets inside the `if (open)` block, add an async IIFE:
```typescript
useEffect(() => {
  if (open) {
    // ... existing synchronous resets (setParseResult(null), etc.) ...
    setSavedResults(null)  // Reset saved results too

    // Async DB check
    ;(async () => {
      setLoadingSaved(true)
      try {
        const saved = await getHPLCAnalysesBySamplePrep(prep.id)
        if (saved.length > 0) {
          setSavedResults(saved)
          const resMap = new Map<string, HPLCAnalysisResult>()
          for (const s of saved) {
            resMap.set(s.peptide_abbreviation ?? '__single__', s)
          }
          setAnalyteResults(resMap)
          const firstKey = resMap.keys().next().value ?? '__single__'
          setResult(resMap.get(firstKey) ?? null)
        }
      } catch {
        // DB check failed — savedResults stays null, SharePoint flow proceeds
      } finally {
        setLoadingSaved(false)
      }
    })()
  }
}, [open, prep.id])
```

This ensures the SharePoint effect (~line 583) fires but immediately bails out when `savedResults` is populated by the DB check. The timing works because:
- Reset effect sets `savedResults = null` synchronously, then starts async DB fetch
- SharePoint effect sees `loadingSaved = true` and bails
- When DB fetch completes: if results found, `savedResults` is set and SharePoint never runs; if no results, `loadingSaved` becomes false and `loadPeakData` can proceed on next render cycle

**Important**: `loadPeakData` dependency array must include `savedResults` and `loadingSaved` so it re-evaluates after the DB check completes.
