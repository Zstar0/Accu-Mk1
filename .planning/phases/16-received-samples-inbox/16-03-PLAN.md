1. **Checkbox** — shadcn Checkbox. Header checkbox: checked when all selected, indeterminate when partial (handle the indeterminate DOM property via ref per Pitfall 4 from RESEARCH.md — if shadcn Checkbox supports a `checked="indeterminate"` prop, use that; otherwise use a ref with useEffect). 2. **Sample ID** — `font-mono text-sm`. Display `sample.id`. Clickable — no navigation in this phase (no external link needed yet, just styled as monospace).
3. **Client** — Display `sample.client_id` or `sample.client_order_number` (whichever is populated, prefer client_order_number).
4. **Priority** — Inline shadcn `Select` component. Options: Normal, High, Expedited. Current value from `sample.priority`. On change → call `onPriorityChange(sample.uid, newPriority)`. Render `PriorityBadge` as the display (the Select trigger should show the badge).
5. **Assigned Tech** — Inline shadcn `Select`. Options populated from `users` prop (display email, value is id). Current value from `sample.assigned_analyst_id`. On change → call `onTechAssign([sample.uid], analystId)`.
6. **Instrument** — Inline shadcn `Select`. Options from `instruments` prop. Current value from `sample.instrument_uid`. On change → call `onInstrumentAssign([sample.uid], instrumentUid)`.
7. **Age** — `<AgingTimer dateReceived={sample.date_received} />`
8. **Status** — `<StateBadge state={sample.review_state} />`
