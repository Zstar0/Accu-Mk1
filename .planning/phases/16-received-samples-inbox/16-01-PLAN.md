**Endpoint 1: GET /worksheets/inbox** (per D-01, D-02)
- Query SENAITE for `sample_received` samples using the existing `list_senaite_samples` pattern (httpx GET to `/senaite/@@API/senaite/v1/AnalysisRequest` with `review_state=sample_received`, `complete=true`)
- Build keyword-to-group map: query `AnalysisService` joined with `service_group_members` joined with `ServiceGroup` — returns dict `{keyword: (group_id, group_name, group_color)}`
- For each sample from SENAITE: attempt to extract analyses from the response. If SENAITE includes analyses inline (check for `Analyses` or `getAnalyses` key in the raw item), use them. Otherwise, for each sample make a separate GET to `/senaite/@@API/senaite/v1/AnalysisRequest/{uid}` to fetch analyses (N+1 pattern — acceptable for <50 samples, log a warning if >20).
- Match each analysis keyword to the group map. Group analyses by service group. Unmatched analyses go to the default group (query `ServiceGroup` where `is_default=True`).
- Load local priorities: `SELECT * FROM sample_priorities WHERE sample_uid IN (list_of_uids)`
- Load worksheet_item assignments: query WorksheetItem for matching sample_uids to get assigned_analyst_id and instrument_uid
- If assigned_analyst_id exists, join to users table to get email for `assigned_analyst_email`
- Return `InboxResponse` with enriched items

**Endpoint 2: PUT /worksheets/inbox/{sample_uid}/priority** (per D-03, D-10)
- Accepts PriorityUpdate body. Validate priority is one of "normal", "high", "expedited"
- Upsert into sample_priorities table (merge/upsert pattern — check if exists, update or insert)
- Return `{"sample_uid": uid, "priority": priority}`

**Endpoint 3: GET /worksheets/users** (per D-05, Pitfall 6 from RESEARCH.md)
- New endpoint accessible to ALL authenticated users (use `Depends(get_current_user)`, NOT `Depends(require_admin)`)
- Query users table for active users only (`is_active=True`)
- Return list of `{"id": int, "email": str}` — do NOT expose hashed_password or SENAITE credentials

**Endpoint 4: PUT /worksheets/inbox/bulk** (per D-16)
- Accepts BulkInboxUpdate body
- For priority: upsert into sample_priorities for each sample_uid
- For analyst_id/instrument_uid: update or create worksheet_items for each sample_uid (these are pre-worksheet assignments — if no worksheet exists yet, store as orphan items that will be picked up when worksheet is created)
- Return `{"updated": len(sample_uids)}`

**Endpoint 5: POST /worksheets** (per D-17, D-18, D-19, D-20)
- Accepts WorksheetCreate body
- **Stale data guard (INBX-10):** For each sample_uid in the request, call SENAITE to verify current review_state is still `sample_received`. Collect any stale UIDs. If any are stale, return HTTP 409 with `{"stale_uids": [list], "message": "N samples have changed state"}`
- If all valid: create Worksheet row, then create WorksheetItem rows for each sample_uid (copy priority from sample_priorities if exists)
- Return the created worksheet: `{"id": int, "title": str, "status": "open", "item_count": int}`
