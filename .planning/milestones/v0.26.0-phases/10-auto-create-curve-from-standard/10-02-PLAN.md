2. Add a new `buildStandardWizardSteps` function below `buildWizardSteps`:
   ```typescript
   /**
    * Builds wizard steps for standard preps: 1 stock + N dilution steps.
    * All dilutions share the ONE stock prep (vial_number=1 for stock measurements).
    * Each dilution has its own vial_number for measurement tracking.
    */
   export function buildStandardWizardSteps(concentrations: number[]): WizardStep[] {
     const steps: WizardStep[] = [
       { id: 1, type: 'sample-info', label: 'Sample Info', vialNumber: 1 },
       { id: 2, type: 'stock-prep', label: 'Stock Prep', vialNumber: 1 },
     ]
     let id = 3
     // Sort concentrations descending (highest first = serial dilution order)
     const sorted = [...concentrations].sort((a, b) => b - a)
     sorted.forEach((conc, i) => {
       steps.push({
         id: id++,
         type: 'standard-dilution',
         label: `Dilution \u2014 ${conc} \u00b5g/mL`,
         vialNumber: i + 1, // Each dilution is a separate physical vial
       })
     })
     return steps
   }
   ```

3. Update `isStepDataComplete` to handle `'standard-dilution'`:
   - Same completion logic as `'dilution'` — checks `actual_conc_ug_ml` in vial calculations.

4. Update `isStepUnlocked` to handle `'standard-dilution'`:
   - First standard-dilution step unlocks when stock prep is complete (vial 1 stock measurements exist).
   - Subsequent standard-dilution steps unlock when the previous step's data is complete (sequential flow — each dilution may depend on the previous for serial dilution).

5. Update `startSession` in the store: when the session has `is_standard === true`, call `buildStandardWizardSteps` instead of `buildWizardSteps`.
