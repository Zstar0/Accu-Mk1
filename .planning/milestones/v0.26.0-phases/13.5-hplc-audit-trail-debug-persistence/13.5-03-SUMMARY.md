## Decisions Made

- **Pre-analysis snapshot over post-analysis patch:** Built `preAnalysisDebugLog` before the `try {}` block so it always reflects actual state regardless of whether analysis succeeds or fails. Avoids needing a PATCH endpoint.
- **Full file content in source_files:** The backend receives the complete raw content alongside the SHA256. This lets the backend verify integrity without re-fetching SharePoint, and enables future diff/replay workflows.
- **`DebugLine[]` cast for DB reload:** `savedResults[0].debug_log` is `{ level: string; msg: string }[] | null` from the API type. Cast to `DebugLine[]` is safe because the backend only ever writes valid level strings — no runtime risk.
- **`useRef` for file archive:** File contents can be large (multiple peak CSVs). Storing in a ref avoids any render cascade while still making the data available when `runAllAnalyses()` fires.
