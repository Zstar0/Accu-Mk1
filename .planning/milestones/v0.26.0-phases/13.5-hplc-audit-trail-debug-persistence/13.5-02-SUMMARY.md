# Phase 13.5 Plan 02: Debug Warning Lines Summary

**buildDebugLines() now emits amber warn lines for 5 silent fallback categories: missing standard injections per blend component, missing chromatogram CSVs, identity RT reference source fallback, missing vial weight data per blend component, and SharePoint download errors**

## Performance

- **Duration:** ~5 min
- **Started:** 2026-03-19T00:00:00Z
- **Completed:** 2026-03-19T00:05:00Z
- **Tasks:** 1
- **Files modified:** 1

## Accomplishments

- Added per-component standard injection check — each blend component without a standard injection now shows an amber warning naming the component and noting identity will fall back to cal curve RT
- Added chromatogram availability section — missing chrom CSV files surface as a warn line for both blend and single-peptide preps
- Added identity reference source display — calibration curve fallback warns in amber, standard injection confirms in green with source ID
- Added per-component vial weight check — blend components with no matching vial number in vial_data emit a warn line with expected vial number
- Added optional `parseError` param to `buildDebugLines()` — SharePoint download errors appear at the top of the debug console before any other output
- Wired `parseError` state from component into the single `buildDebugLines()` call site

## Task Commits

Each task was committed atomically:

1. **Task 1: Add warning lines to buildDebugLines() for silent fallbacks** - `72b1ca9` (feat)

## Files Created/Modified

- `src/components/hplc/SamplePrepHplcFlyout.tsx` - Added 45 lines of warning logic across 5 categories; added optional `parseError` param; updated call site

## Decisions Made

- Per-component standard injection check iterates `labelToComponent` entries (not raw `blendComponents`) so only resolved/matched labels are checked — unmatched labels already have error lines from the mapping section
- Chromatogram availability section is not gated on `isBlend` — single-peptide preps can also be missing chrom files and deserve the warning
- Identity reference source warning is placed after the formulas section (still inside the `if (currentResult?.calculation_trace)` block) so it appears right after the identity RT formula context
- Vial weight missing-component check only activates when `prep.vial_data` is non-empty — if there's no vial data at all, the existing "No per-vial weight data" warning already covers the broader case
