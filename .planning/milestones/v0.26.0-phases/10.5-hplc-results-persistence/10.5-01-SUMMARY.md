## Decisions Made
- `sample_prep_id` is plain INTEGER with no FK — sample_preps lives in a separate database (accumark_mk1), cross-database FK is impossible
- `calibration_curve_id` is persisted from `cal.id` resolved during analysis, not from request input — guarantees the stored curve ID always matches the curve that was actually used
- Registered `/hplc/analyses/by-sample-prep/{sample_prep_id}` before `/{analysis_id}` routes — FastAPI matches routes in registration order; without this, the literal path segment "by-sample-prep" would be parsed as an integer and 422 would be returned
