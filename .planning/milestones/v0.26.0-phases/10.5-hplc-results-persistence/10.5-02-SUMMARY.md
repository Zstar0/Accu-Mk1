## Decisions Made

- `instrument_id` passed as `undefined` throughout — SamplePrep interface lacks this field (only `instrument_name` exists). Comment left in code indicating how to update when backend exposes it.
- `chromatogram_data` is derived from `chromTraces[0]` only — for both single and blend runs the injections share one physical chromatogram file, so the first trace is the authoritative one.
- Status update is fire-and-forget inside a try/catch — analysis results are already stored server-side; a status update failure must not surface as an analysis failure.
- `loadingSaved` guard handles the timing window where the SharePoint effect fires before the async DB check completes.
