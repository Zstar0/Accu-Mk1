## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline chip padding (px-1 py-0.5) |
| sm | 8px | Between items in a list row, badge internal gaps |
| md | 16px | Drawer section padding (p-4), between header fields |
| lg | 24px | Between drawer header and items list |
| xl | 32px | FAB bottom offset from screen edge |
| 2xl | 48px | Major section breaks within drawer scroll area |
| 3xl | 64px | — (not used in Phase 17) |

Exceptions:
- FAB: `bottom-8 right-4` (32px bottom, 16px right) — fixed position, not part of flow layout
- FAB touch target: h-12 w-12 (48px) — minimum accessible touch target per WCAG 2.5.5
- Item row remove button: h-8 w-8 (32px) with hover zone — compact but accessible
- Sheet content: `p-0` on SheetContent, `px-4 py-3` on internal sections — overrides shadcn default padding for density

## Color

All values are CSS custom properties from theme-variables.css (light/dark dual mode).

| Role | Light | Dark | Usage |
|------|-------|------|-------|
| Dominant (60%) | `--background` oklch(1 0 0) = #ffffff | oklch(0.145 0 0) = ~#1a1a1a | Sheet backdrop, drawer body background |
| Secondary (30%) | `--card` oklch(1 0 0) / `--muted` oklch(0.97 0 0) | oklch(0.205 0 0) / oklch(0.269 0 0) | Worksheet header section bg, item rows bg on hover, section dividers |
| Accent (10%) | `--primary` oklch(0.205 0 0) = ~#1a1a1a | oklch(0.87 0 0) = ~#dedede | See reserved list below |
| Destructive | `--destructive` oklch(0.58 0.22 27) = ~#d4382a | oklch(0.704 0.191 22.216) = ~#f06040 | Remove-item button hover, delete worksheet confirm |

Accent reserved for (explicit list — not general interactive):
1. FAB button background (`bg-primary text-primary-foreground`)
2. FAB item badge counter (`bg-destructive` for item count — red badge, not primary)
3. "Complete Worksheet" button (primary variant — only one per drawer, high-intent)
4. Active worksheet tab underline/indicator when multiple tabs shown
5. Drag-over state on drop zones (`border-primary bg-primary/5`)
6. Inline title save/confirm button (check icon, `text-primary`)

Status badge colors (semantic, not accent):
- `open`: `bg-emerald-100 text-emerald-800 border-emerald-200` / dark: `bg-emerald-900/30 text-emerald-300`
- `completed`: `bg-zinc-100 text-zinc-600 border-zinc-200` / dark: `bg-zinc-900/30 text-zinc-400`
- `cancelled`: `bg-red-100 text-red-700 border-red-200` / dark: `bg-red-900/30 text-red-300`

Service group badge colors: sourced from `SERVICE_GROUP_COLORS` in `src/lib/service-group-colors.ts` — 8 options (blue, amber, emerald, red, violet, zinc, rose, sky), applied via `colorKey` property on each service group.

Priority badge colors:
- `normal`: zinc-100/zinc-700 (light), zinc-900/30/zinc-300 (dark)
- `high`: amber-100/amber-800 (light), amber-900/30/amber-300 (dark)
- `expedited`: red-100/red-800 + animate-pulse (light), red-900/30/red-300 (dark)

## Layout Specification

### FAB (Floating Action Button)

- Position: `fixed bottom-8 right-4` (32px from bottom, 16px from right)
- Size: `h-12 w-12` (48px — accessible touch target)
- Shape: `rounded-full`
- Z-index: `z-40` (above page content, below Sheet overlay at z-50)
- Background: `bg-primary text-primary-foreground shadow-lg`
- Hover: `hover:bg-primary/90 transition-colors`
- Icon: `ClipboardList` from lucide-react, `h-5 w-5`
- Badge: absolute `-top-1 -right-1`, `h-5 w-5 rounded-full bg-destructive text-destructive-foreground text-[10px] font-semibold`
- Badge content: total open item count across all non-completed worksheets (see Open Question resolution below)
- When drawer is open: FAB hidden (Sheet overlay covers it; Tauri desktop — no mobile back-button concern)
- aria-label: "Open worksheet"

### Focal Point

Primary focal point: FAB button (`bg-primary`, fixed bottom-right). Inside the drawer: worksheet title (text-base, weight 600) and the "Complete Worksheet" button (primary variant, right-aligned in action row).

### Sheet Drawer

- Side: `right`
- Width: `w-[480px] sm:max-w-[480px]` (480px fixed — enough for item table columns)
- Padding: `p-0` on SheetContent; sections use internal `px-4 py-3`
- Z-index: shadcn default `z-50` (via Radix Dialog portal)
- Backdrop: shadcn default (semi-transparent overlay, click-outside closes)
- Animation: shadcn default slide-in-from-right (tw-animate-css handles this)

### Drawer Internal Layout (top to bottom)

```
[SheetHeader — px-4 py-3 border-b]
  Worksheet title (inline-editable h1)
  Status badge + item count badge
  Created date (text-xs muted-foreground)

[Worksheet tabs — only shown when 2+ worksheets exist]
  shadcn Tabs — one tab per open worksheet, scrollable if many

[Header fields — px-4 py-3 border-b bg-muted/30]
  Tech dropdown (full width)
  Notes textarea (min-h-[72px], expandable)

[Action row — px-4 py-2 border-b flex gap-2]
  "Add Samples" button (secondary/outline variant)
  "Complete Worksheet" button (primary variant, right-aligned)

[Items section — flex-1 overflow-hidden]
  "Items (N)" label — px-4 pt-3 pb-1 text-xs font-semibold muted-foreground
  ScrollArea — full remaining height
    Item rows (see below)
    Empty state (when no items)
```

### Item Row Layout

Reuses WorksheetDropPanel item row pattern (lines 172-195):

```
[remove X button] [sample ID mono] [·] [group badge] [priority badge] [age timer] [reassign button] [Start Prep button]
```

- Height: auto, min 28px per row
- `group/item` hover reveals remove button (opacity-0 → opacity-100)
- Group badge: `inline-flex rounded-md border px-2 py-0.5 text-xs font-semibold` using `SERVICE_GROUP_COLORS[colorKey]`
- "Start Prep" button: `h-6 text-[10px]` secondary variant, visible on row hover; shows "Prep started" state as `text-[10px] text-muted-foreground/60 italic` once triggered
- "Reassign" button: icon-only (`MoveRight` lucide, h-3 w-3), opens a popover with worksheet picker; aria-label: "Move {sample_id} to another worksheet"

## Interaction Contracts

### Inline Title Edit

- Trigger: click on title text → replaces with `Input` (h-7, text-base)
- Save: Enter key or click checkmark button (`Check` icon, `text-primary`)
- Cancel: Escape key or click X button (`text-muted-foreground`)
- API: `PUT /worksheets/{id}` with `{ title }` — optimistic update, revert on error
- Validation: trim whitespace; reject empty string (restore previous value)

### Notes Field

- Component: `Textarea` with `min-h-[72px]`
- Save trigger: onBlur (auto-save when focus leaves) — no explicit save button
- Debounce: 800ms after last keystroke before API call
- API: `PUT /worksheets/{id}` with `{ notes }` — no optimistic update (fire-and-forget on blur)
- Placeholder: "Add notes…"

### Tech Assignment Dropdown

- Component: `Select` (full-width, h-8)
- Options: all users from `GET /users` (cached, staleTime: 5 minutes)
- Placeholder: "Assign tech…"
- API: `PUT /worksheets/{id}` with `{ assigned_analyst: userId }` on change

### Remove Item

- Trigger: click X button on item row (revealed on hover)
- No confirmation required (reversible — item returns to inbox)
- API: `DELETE /worksheets/{id}/items/{uid}/{gid}`
- Behavior: item disappears immediately (optimistic remove); toast "Item removed — now back in inbox"
- Keyboard: button must be focusable; aria-label: "Remove {sample_id} from worksheet"

### Reassign Item

- Trigger: click `MoveRight` icon on item row (revealed on hover)
- Opens: `Popover` (not a modal) with a `Select` listing other open worksheets
- API: `POST /worksheets/{id}/items/{uid}/{gid}/reassign` with `{ target_worksheet_id }`
- Behavior: item disappears from current worksheet items list; toast "Item moved to {target title}"
- If no other open worksheets: button is disabled with tooltip "No other open worksheets"

### Add Samples

- Trigger: "Add Samples" button
- Opens: `AddSamplesModal` — a shadcn `Dialog` (not a Sheet)
- Content: simplified inbox card list, filtered to items not already in any worksheet
- Interaction: click a card to add it to the current worksheet (no DnD — avoids DnD context outside WorksheetsInboxPage)
- API: `POST /worksheets/{id}/add-group` per item added
- On add: item card shows checkmark, stays visible (user may add more); close button dismisses modal
- Empty state: "No unassigned samples in the inbox."

### Complete Worksheet

- Trigger: "Complete Worksheet" button (primary variant, right-aligned in action row)
- Opens: `AlertDialog` confirmation
  - Title: "Complete this worksheet?"
  - Description: "This worksheet will be marked as completed and removed from the active queue. This cannot be undone."
  - Cancel: "Keep Worksheet" (outline — keeps the worksheet active)
  - Confirm: "Complete Worksheet" (destructive variant — irreversible action)
- API: `POST /worksheets/{id}/complete`
- On success: drawer closes, `['worksheets']` query invalidated, toast "Worksheet completed"
- Badge: completed worksheets excluded from FAB item count

### Start Prep

- Trigger: "Start Prep" button per item row (secondary, h-6, text-[10px])
- Behavior: stores prefill in `ui-store.worksheetPrepPrefill`, navigates to `hplc-analysis/new-analysis`, closes drawer
- Prefill data: `{ sampleId, peptideId, method }` derived from item's service group
- "Prep started" indicator: shown on item row after button is clicked for this session; state persisted in `WorksheetItem.notes` as JSON flag
- Multiple preps allowed per item — indicator is informational only, button remains clickable

### Worksheet Tabs (D-05)

- Shown only when 2+ worksheets are open (status = 'open')
- Component: shadcn `Tabs` — one tab per worksheet
- Tab label: worksheet title (truncated at 20 chars) + item count badge
- Active tab: sets `activeWorksheetId` in Zustand
- Overflow: `overflow-x-auto` on tab list if many worksheets

### Hash Navigation

- Route `#hplc-analysis/worksheet-detail?id=X` parses `id`, calls `openWorksheetDrawer(Number(id))`, does NOT change `activeSection`/`activeSubSection`
- FAB click does NOT produce a hash change — drawer open state is Zustand-only
- Browser back after hash-driven open: drawer closes (hash no longer matches)

## States

### Drawer Loading State

- Show 3 skeleton rows (animate-pulse) in the items section
- Header fields show skeleton inputs
- Duration: until `['worksheets']` query resolves

### Empty Items State

- Shown when worksheet has 0 items
- Icon: `ClipboardX` from lucide-react, `h-8 w-8 text-muted-foreground/40`
- Heading: "No items in this worksheet"
- Body: "Add samples from the inbox to get started."
- CTA: "Add Samples" button (same as header action row button)

### Error State (drawer data fetch failure)

- Show `Alert` (variant destructive) inside drawer
- Heading: "Could not load worksheet"
- Body: "Check your connection and try again."
- Action: "Retry" button that calls `refetch()`

### Completed Worksheet View

- Drawer opens in read-only mode for completed worksheets
- Header shows "Completed" status badge (zinc variant)
- Title, notes, tech are rendered as static text (no inline edit)
- Items are listed but remove/reassign/start-prep actions are hidden
- Action row shows only: "View only — worksheet is completed" muted text
- "Complete Worksheet" button hidden

| Question | Resolution |

| FAB when drawer is open | Hidden (covered by Sheet overlay). No explicit hide logic needed for desktop Tauri app. |
