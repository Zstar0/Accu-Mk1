  <action>
Add a diagnostic endpoint `POST /senaite/analyses/{uid}/analyst-test` that tests both username and UID formats against a live SENAITE instance. This endpoint:

1. Accepts `{"username": "lab_tech_1", "uid": "abc123uid"}` (both the username string and the analyst's UID from LabContact)
2. First reads the current Analyst value: `GET {SENAITE_URL}/senaite/@@API/senaite/v1/{uid}` — store current value
3. Tries setting with username: `POST {SENAITE_URL}/senaite/@@API/senaite/v1/update/{uid}` with `{"Analyst": username}` — reads back the stored value
4. Tries setting with UID: `POST {SENAITE_URL}/senaite/@@API/senaite/v1/update/{uid}` with `{"Analyst": uid}` — reads back the stored value
5. Restores the original value
6. Returns a diagnostic response:
```json
{
  "original_value": "...",
  "username_test": {"sent": "lab_tech_1", "stored": "...", "accepted": true/false},
  "uid_test": {"sent": "abc123uid", "stored": "...", "accepted": true/false},
  "recommendation": "use_username" | "use_uid" | "unclear"
}
```

The "accepted" flag is `true` if the stored value matches the sent value OR if the stored value is non-null after the update (SENAITE may transform the value).

Use the same `httpx.AsyncClient` + `_get_senaite_auth(current_user)` + `Timeout(30.0, connect=5.0)` pattern as existing SENAITE endpoints. Guard with `if not SENAITE_URL: raise HTTPException(503, "SENAITE URL not configured")`.

Pydantic schema:
```python
class AnalystTestRequest(BaseModel):
    username: str
    uid: str
```
